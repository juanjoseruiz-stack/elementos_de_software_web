1. Conceptos de Internet
a. Servidor DNS
n servidor DNS (Sistema de Nombres de Dominio) es un sistema fundamental en el funcionamiento de internet que se encarga de traducir los nombres de dominio que las personas usan (como google.com) en direcciones IP que las computadoras entienden. Cuando una persona escribe una dirección web en su navegador, el servidor DNS busca la dirección IP correspondiente en su base de datos y permite que el navegador se conecte con el servidor correcto. Este proceso ocurre en segundos y facilita el uso de internet, ya que evita que los usuarios tengan que memorizar números largos y complejos.

b. IP

Una dirección IP (Protocolo de Internet) es un número único que identifica a cada dispositivo conectado a una red. Funciona como una especie de “dirección” que permite ubicar un equipo dentro de internet. Gracias a la IP, los dispositivos pueden enviarse información entre sí. Existen dos versiones principales: IPv4, que es la más común y usa números separados por puntos, e IPv6, que es más moderna y permite una mayor cantidad de direcciones.

c. Protocolo

Un protocolo es un conjunto de reglas y normas que permiten la comunicación entre dispositivos dentro de una red. Estas reglas indican cómo se envían, reciben y procesan los datos. Por ejemplo, el protocolo HTTP permite la carga de páginas web, mientras que HTTPS añade una capa de seguridad para proteger la información. Sin los protocolos, los dispositivos no podrían entenderse entre sí ni intercambiar información correctamente.

d. Nombre de dominio

Un nombre de dominio es la dirección que las personas escriben en el navegador para acceder a un sitio web. Es una forma fácil de recordar la ubicación de una página en internet, en lugar de usar una dirección IP numérica. Los nombres de dominio están conectados a direcciones IP mediante el sistema DNS. Por ejemplo, direcciones como youtube.com o wikipedia.org permiten acceder a páginas específicas de manera sencilla.

e. Directorio y nombre de archivo

Un directorio es una carpeta que sirve para organizar archivos dentro de un sistema o una página web. El nombre de archivo identifica un archivo específico dentro de ese directorio. Por ejemplo, en una ruta web como “/imagenes/foto.jpg”, “imagenes” es el directorio y “foto.jpg” es el archivo. Esta estructura permite mantener el contenido ordenado y facilita la localización de la información.

2. Roles en desarrollo web
a. Diseñador UI/UX

El diseñador UI/UX se encarga de diseñar la apariencia visual de una página web (interfaz de usuario) y de mejorar la experiencia del usuario al interactuar con ella. Este profesional define elementos como colores, botones, tipografías y organización del contenido. Además, analiza cómo navegan los usuarios dentro del sitio para hacerlo más intuitivo, fácil de usar y atractivo.

b. Cloud Manager / Ingeniero en la nube

El ingeniero en la nube es el responsable de gestionar los recursos tecnológicos que funcionan a través de internet, como servidores, almacenamiento y aplicaciones. Se encarga de configurar plataformas en la nube, asegurar que el sistema funcione correctamente y mantener la seguridad de los datos. También optimiza el rendimiento del sitio web para que cargue rápido y funcione sin fallos.

c. Administrador de base de datos

El administrador de base de datos organiza, almacena y protege la información que utiliza una página web o aplicación. Su trabajo consiste en asegurar que los datos estén disponibles, sean seguros y se puedan consultar de manera rápida. También realiza copias de seguridad y evita la pérdida de información importante.

d. Project Manager / Director del Proyecto

El project manager es la persona encargada de planificar, coordinar y supervisar todo el desarrollo del proyecto web. Define los objetivos, organiza el trabajo del equipo, establece tiempos de entrega y asegura que se cumplan los resultados esperados. Además, toma decisiones importantes y resuelve problemas que surgen durante el proceso.

3. Lenguaje Turing completo

Un lenguaje es Turing completo cuando tiene la capacidad de resolver cualquier problema computacional, siempre que disponga de suficiente tiempo y recursos. Esto significa que puede realizar cálculos complejos, usar estructuras como condiciones (if), ciclos (loops) y manejar datos de manera avanzada. Los lenguajes de programación más utilizados cumplen con esta característica.

4. Por qué CSS y HTML no son lenguajes de programación

HTML y CSS no son considerados lenguajes de programación porque no tienen la capacidad de ejecutar lógica ni tomar decisiones. HTML se utiliza para estructurar el contenido de una página web, como textos, imágenes y enlaces, mientras que CSS se encarga del diseño visual, como colores, tamaños y estilos. Ninguno de los dos permite realizar procesos complejos o cálculos como lo hacen los lenguajes de programación.

5. Qué es una etiqueta en HTML

Una etiqueta en HTML es un elemento que se utiliza para definir la estructura de una página web. Las etiquetas indican al navegador cómo debe mostrarse el contenido. Existen diferentes tipos de etiquetas, como las de títulos, párrafos, imágenes y enlaces. Generalmente, las etiquetas tienen una apertura y un cierre, lo que permite delimitar el contenido que contienen y organizar correctamente la información en la página.
